# CORE Role Guide

**Welcome to Operations Tools!** This guide covers everything available to users with the CORE role.

## Your Sidebar

After logging in, you'll see a sidebar on the left with the following sections:

- **Core Tools** — Alignment Scoring, Likert Scoring, Task Search, Similarity Flags
- **QA Tools** — Records, Similarity Search, Top Prompts, Top/Bottom 10
- **Resources** — Links

Some items open in the same tab; others (marked with a small external link icon) navigate to a different part of the platform. Either way, your session stays active and you won't need to log in again.

**Searching the sidebar:** Type in the **Search tools...** input at the top of the sidebar to filter all navigation items by name. Matching results appear in a flat list with their section shown beneath each label. Clear the search or press the collapse button to return to the normal view.

**Similarity Flags badge:** A warning icon (⚠) in the header shows the count of open similarity flags. Click it to go directly to the Similarity Flags dashboard. The badge is visible from any app while you are logged in with a CORE or higher role.

## Alignment Scoring

Browse records and view their AI-generated alignment analysis scores to assess how well tasks and feedback align with project guidelines.

> **Note**: Alignment Scoring is currently in testing and not yet fully implemented. You may encounter functionality errors or incomplete data while using this feature.

### What is Alignment Scoring?

Alignment Scoring shows records that have been analyzed by AI against project guidelines. Each record has a numeric score (1–10) extracted from the analysis, along with the full analysis text covering strengths, weaknesses, and suggestions.

This is a **read-only view** — you browse and review scores that the system has already generated. It helps you identify high- and low-quality records at a glance.

### Filtering Records

Use the filter controls at the top to narrow down what you're viewing:

**Environment**: Select from the dropdown to filter to a specific data batch. Leave blank to view all.

**Type**:
- **TASK** — Prompt/task records
- **FEEDBACK** — Feedback records

**Category**:
- **ALL** — All records (default)
- **TOP_10** — Records flagged as top-quality
- **BOTTOM_10** — Records flagged as lowest-quality
- **STANDARD** — Regular records

### Reading the Results

Records are shown 10 per page. Each card displays:
- The record content
- Type and category badge
- **Alignment score** (1–10)
- The full alignment analysis with strengths, weaknesses, and suggestions

Use the page navigation at the bottom to move through results.

### Tips

- Use **BOTTOM_10** + a specific environment to quickly find records that need attention
- Records without an alignment analysis will show no score — analysis must be generated by a Fleet-level user first

## Likert Scoring

Rate task and feedback records on standardized quality dimensions to provide quantitative assessments used in performance tracking and quality analysis.

### What is Likert Scoring?

You rate individual records on two dimensions using 1–7 scales:

- **Realism** (1–7): How realistic and believable is the content?
- **Quality** (1–7): What is the overall quality of the content?

Your ratings are submitted and stored. The system tracks which records you have already scored so you won't be shown the same record twice.

### Scale Reference

| Score | Meaning |
|-------|---------|
| 1 | Very poor |
| 2 | Poor |
| 3 | Below average |
| 4 | Average / acceptable |
| 5 | Above average |
| 6 | Good |
| 7 | Excellent |

### Step-by-Step Workflow

**1. Select an Environment**

You must select an environment before any records are shown. Use the dropdown at the top to choose which data batch you're working on.

**2. View a Record**

The system shows you unscored records from the **TOP_10** and **BOTTOM_10** categories. Each record displays the full content, type, category badge, and creator metadata.

**3. Rate the Record**

Click the appropriate number on each scale — one score for **Realism**, one for **Quality**. The selected score is highlighted.

**4. (Optional) Run LLM Evaluation**

Each record has an **LLM Evaluate** button that sends it to the AI for an automated quality evaluation. Use this when you're unsure or want a structured second opinion before submitting your own rating.

**5. Submit Your Rating**

Click **Submit** to save your scores. The record is marked as scored, and the next unscored record loads automatically.

### Best Practices

- **Be consistent**: Use the same mental model for each number across all records
- **Read the full content**: Don't skim — the full text matters for both dimensions
- **Take breaks**: Scoring fatigue leads to drift; step away every 20–30 records
- **Use LLM Evaluate**: If a record seems borderline, run the evaluation for a structured second opinion

## Task Search

Search for tasks by creator name, email, or task ID. Use the built-in AI authenticity checker to assess whether a task was written authentically or generated by AI.

### Searching

Type in the search box using any of:
- **Creator name** (first name, last name, or partial)
- **Creator email** (full or partial)
- **Task ID** (exact or partial)

The **Search** button is disabled until you type something. Press **Enter** or click **Search** to run the query.

A result count is shown at the top: e.g., **"24 results found"**

### Environment Filter

When results span **more than one environment**, a dropdown filter appears above the results. Use it to narrow down to a specific environment.

When a filter is active, the count updates to: **"12 of 24 results in 'environment-name'"**

If all results are from a single environment, the filter is not shown.

### AI Authenticity Check

Each result card has an **AI Check** button. Click it to run an AI analysis assessing how the task was likely written.

The button shows **"Checking…"** while the analysis runs (a few seconds), then the result panel appears with:

- **Verdict** — one of three outcomes:
  - **Authentic** — Appears to be written genuinely by a human
  - **AI Generated** — Likely produced by an AI model
  - **Templated** — Appears to follow a fill-in-the-blank template
- **Confidence** — HIGH, MEDIUM, or LOW
- **Reasoning** — The AI's explanation for its verdict
- **Indicators** — Specific signals in the text that informed the verdict

### Tips

- HIGH confidence verdicts are reliable; LOW confidence means the task is ambiguous
- "Templated" often indicates copy-paste patterns or repeated structures
- A single check is a signal, not a final judgment — context matters
- Run checks on tasks that look identical or suspiciously similar to others

## Similarity Flags

Review prompt pairs that the system automatically detected as highly similar during data ingestion. These flags are generated without manual intervention — whenever a new batch of tasks is ingested and vectorized, the system compares each new prompt against historical prompts from the same user and creates a flag for any pair exceeding the similarity threshold (default: 80%).

**Who can access this page**: CORE, FLEET, MANAGER, ADMIN roles.

### What is a Similarity Flag?

Each flag represents two task records — a source (the newly ingested task) and a match (a previously ingested task from the same user) — where the cosine similarity of their vector embeddings exceeds the threshold. A high similarity score means the two prompts are semantically very close and may warrant investigation.

Flags are not a verdict. They are a signal for a reviewer to look at. Only a human review can determine whether the similarity indicates a problem.

### Dashboard Overview

The Similarity Flags page displays a table of detected pairs with the following columns:

- **Status** — Either Open (not yet reviewed) or Claimed (a reviewer has indicated they are investigating)
- **User** — The name and email of the worker who submitted both records
- **Score** — The cosine similarity expressed as a percentage (e.g., 94.3%)
- **Source (snippet)** — A truncated preview of the newly ingested task prompt
- **Match (snippet)** — A truncated preview of the historically matched prompt
- **Environment** — The data batch the source record belongs to
- **Date** — When the flag was created

### Filtering Flags

Use the controls above the table to narrow down what you see:

- **Status tabs** — Switch between All, Open, and Claimed views
- **Mine only** — Appears when the Claimed tab is active; restricts the list to flags you personally claimed
- **Environment dropdown** — Filter to a specific data batch

Click **Refresh** to reload the current view without changing your filter settings.

### Claiming a Flag

When you decide to investigate a flag, click the **Claim** button in the actions column. Claiming records that you are investigating prevents multiple reviewers from duplicating effort. Claimed flags move to the Claimed tab and show your email address.

Claiming does not resolve or close a flag — it is a lightweight coordination signal only.

### Viewing Full Record Details

Click any snippet text in the Source or Match columns to open a detail modal for that record. The modal shows:

- The full prompt text
- Task ID (if available in the record metadata)
- Created by (name and email)
- Environment
- Date

For source records that have a task ID, a **View in Task Search** link appears in the modal header. Clicking it opens the Task Search page pre-populated with that task ID, giving you access to the full AI authenticity check workflow.

Close the modal by clicking the **×** button in the top-right corner or clicking outside the modal.

## Reporting Bugs

A bug report button is available on every page — look for the bug icon in the bottom corner of the screen. Click it to open a form where you can describe what you were doing when the issue occurred. The current page URL is captured automatically.

Bug reports are reviewed by operations staff. You don't need to include any technical details — just describe the action that led to the problem.

## Troubleshooting

### "Likert Scoring shows no records after selecting environment"

You have already scored all TOP_10/BOTTOM_10 records in that environment. Either choose a different environment or wait for new data to be ingested.

### "AI Check button shows an error"

The AI service may be temporarily unavailable. Wait a moment and try again. Contact your manager if the issue persists.

### "Task Search returns no results"

- Try a broader search (just a first name, or just the domain of their email)
- Verify the spelling — search is not fuzzy
- The creator may not have any records in the system yet

### "Alignment scores are missing from records"

Alignment analysis must be run separately by a Fleet-level user. Contact your manager to request analysis be generated for those records.

### "I don't see a section in the sidebar"

Your role may not include access to that section. Contact your manager if you believe you need access.
